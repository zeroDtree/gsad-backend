```
│   │   ├── server/      # api(public_, internal) · model · persistence · service(ServerMetricsReader)
```

## 3. 数据库设计

### 3.1 用户表 `t_user`

```sql
CREATE TABLE t_user (
    id           BIGSERIAL       PRIMARY KEY,
    email        VARCHAR(255)    NOT NULL UNIQUE,
    password     VARCHAR(255)    NOT NULL,            -- BCrypt 哈希
    roles        VARCHAR(255)    NOT NULL DEFAULT '',  -- 逗号分隔，如 "admin"
    created_at   TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);
```

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGSERIAL | 主键 |
| `email` | VARCHAR(255) | 唯一，用作登录名 |
| `password` | VARCHAR(255) | BCrypt 哈希存储 |
| `roles` | VARCHAR(255) | 逗号分隔角色串，如 `"admin"` 或 `""` |

### 3.2 服务器表 `t_server`

```sql
CREATE TABLE t_server (
    id                 BIGSERIAL       PRIMARY KEY,
    hostname           VARCHAR(255)    NOT NULL UNIQUE,
    resource_level     VARCHAR(50)     NOT NULL,       -- H100/A100/L40S/L4/T4
    status             VARCHAR(20)     NOT NULL DEFAULT 'OFFLINE',
    last_reported_at   TIMESTAMPTZ,
    avg_util           DECIMAL(5,4),                  -- 0.0000~1.0000
    avg_mem_used_mb    INTEGER,
    created_at         TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at         TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);
```

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `hostname` | VARCHAR(255) | 服务器唯一标识 |
| `resource_level` | VARCHAR(50) | 资源等级枚举 |
| `status` | VARCHAR(20) | `ONLINE` / `OFFLINE` / `MAINTENANCE` |
| `avg_util` | DECIMAL(5,4) | GPU 平均利用率，0~1 范围 |
| `avg_mem_used_mb` | INTEGER | GPU 平均显存占用（MB） |

### 3.3 申请表 `t_application`

```sql
CREATE TABLE t_application (
    id                   VARCHAR(64)     PRIMARY KEY,   -- 业务 ID，如 "app-abc123"
    user_id              BIGINT          NOT NULL REFERENCES t_user(id),
    user_email           VARCHAR(255)    NOT NULL,      -- 冗余字段，避免 JOIN
    resource_level       VARCHAR(50)     NOT NULL,
    purpose              VARCHAR(500)    NOT NULL,
    requested_days       INTEGER         NOT NULL CHECK (requested_days > 0),
    requested_start_at   TIMESTAMPTZ     NOT NULL,
    expire_at            TIMESTAMPTZ,
    audit_status         VARCHAR(20)     NOT NULL DEFAULT 'APPROVED',
    comment              TEXT,
    idempotency_key      VARCHAR(64)     UNIQUE,        -- 防重复提交
    created_at           TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at           TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_application_user_id ON t_application(user_id);
CREATE INDEX idx_application_audit_status ON t_application(audit_status);
CREATE INDEX idx_application_updated_at ON t_application(updated_at DESC);
```

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | VARCHAR(64) | 业务 ID，使用 `"app-" + UUID` 前8位 |
| `user_id` | BIGINT | 外键关联用户 |
| `user_email` | VARCHAR(255) | 冗余，便于列表与日志展示 |
| `audit_status` | VARCHAR(20) | 状态机值，详见第4节 |
| `idempotency_key` | VARCHAR(64) | 唯一约束，防重复提交 |

## 4. 业务状态机

### 4.1 `audit_status` 枚举

```java
public enum AuditStatus {
    APPROVED,      // 已提交，授权任务执行中
    ACTIVE,        // 授权成功，资源已生效
    EXPIRED,       // 已到期，资源已回收
    FAILED_GRANT, // 授权任务失败
    FAILED_REVOKE  // 回收任务失败
}
```

### 4.2 状态转移规则

```
提交申请 ──►  APPROVED  ──[provision/complete 成功]──►  ACTIVE  ──[revoke/complete 成功]──►  EXPIRED
                      └──[provision/complete 失败]──►  FAILED_GRANT
                                            ACTIVE  ──[revoke/complete 失败]──►  FAILED_REVOKE
```

### 4.3 实现说明

- 创建申请时：`ApplicationService` 解析 Linux 用户名与密码（gsad 提供），保存为 `APPROVED`，**不**触发出站调用。
- 外部开账号服务通过 `POST .../report` 拉取 `pendingGrants` / `pendingRevokes`，本地执行开/删号后回调 `provision/complete` 或 `revoke/complete`。
- 凭据（`ssh_username`、`password`）由 gsad 在 create 时确定；complete 仅回报 `success` 与 `serverIp`。
- 到期 `ACTIVE` 申请出现在 `pendingRevokes` 中，由外部服务回收（`ExpirationScheduler` 仅打 debug 日志）。

### 5.3 全局异常处理

```java
// com.zerodtree.gsad.common.GlobalExceptionHandler
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ApiResponse<Void>> handleBusiness(BusinessException ex) {
        HttpStatus status = switch (ex.getErrorCode()) {
            case INVALID_ARGUMENT        -> HttpStatus.BAD_REQUEST;
            case UNAUTHORIZED            -> HttpStatus.UNAUTHORIZED;
            case FORBIDDEN               -> HttpStatus.FORBIDDEN;
            case NOT_FOUND               -> HttpStatus.NOT_FOUND;
            case STATE_CONFLICT          -> HttpStatus.CONFLICT;
            case RATE_LIMITED            -> HttpStatus.TOO_MANY_REQUESTS;
        };
        return ResponseEntity.status(status)
            .body(ApiResponse.error(ex.getErrorCode(), ex.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiResponse<Void>> handleValidation(MethodArgumentNotValidException ex) {
        String msg = ex.getBindingResult().getFieldErrors().stream()
            .map(f -> f.getField() + ": " + f.getDefaultMessage())
            .collect(Collectors.joining("; "));
        return ResponseEntity.badRequest()
            .body(ApiResponse.error(ErrorCode.INVALID_ARGUMENT, msg));
    }
}
```

## 6. 接口详细设计

### 6.1 用户认证模块（`UserController`）

#### POST `/api/auth/register`

**无需鉴权**

```
请求体：RegisterRequest
  - email    : @Email @NotBlank
  - password : @Size(min=8) @NotBlank

处理逻辑：
  1. 检查 email 是否已存在 → 409 STATE_CONFLICT
  2. BCryptPasswordEncoder 加密密码
  3. 保存 User（roles 为空）
  4. 签发 JWT
  5. 返回 201 Created + {token, email, roles}
```

#### POST `/api/auth/login`

**无需鉴权**

```
请求体：LoginRequest
  - email    : @Email @NotBlank
  - password : @NotBlank

处理逻辑：
  1. 按 email 查询用户 → 不存在则 401 UNAUTHORIZED
  2. BCrypt 比对密码 → 不匹配则 401 UNAUTHORIZED
  3. 签发 JWT（payload 含 email、roles、exp）
  4. 返回 200 OK + {token, email, roles}
```

### 6.2 公开服务器接口（`ServerController`）

#### GET `/api/public/servers`

**无需鉴权**，返回全部服务器及 GPU 汇总

```
处理逻辑：
  1. 查询 t_server 全表（小数据量，无需分页）
  2. 组装响应：hostname, resource_level, status,
               last_reported_at, gpu_summary{avg_util, avg_mem_used_mb}
  3. 返回 200 OK + {items: [...]}

注意：avg_util 存储为 0~1，前端直接展示百分比时自行处理
```

**响应 DTO：**
```java
public record ServerVO(
    String hostname,
    String resourceLevel,
    String status,
    Instant lastReportedAt,
    GpuSummaryVO gpuSummary
) {}

public record GpuSummaryVO(
    BigDecimal avgUtil,      // nullable
    Integer avgMemUsedMb     // nullable
) {}
```

### 6.3 申请模块（`ApplicationController`）

#### POST `/api/applications`

**需要 JWT**，防重复提交

```
请求头：Idempotency-Key: <UUID v4>
请求体：CreateApplicationRequest（Jackson camelCase）
  - serverId          : @NotBlank（来自 GET /api/public/servers 的 id）
  - purpose           : @NotBlank @Size(max=500)
  - requestedDays     : @Min(1)
  - requestedStartAt  : @NotNull（UTC ISO 8601）
  - sshPassword       : 可选

处理逻辑：
  1. `@CurrentUserId` 获取当前用户
  2. 读取 `Idempotency-Key` header（前端每次提交带 UUID）
  3. 若 key 存在：先查 Redis，再查 `t_application.idempotency_key`，命中则返回原记录
  4. 校验 `serverId` 对应服务器存在，复制 `resource_level`
  5. 解析 `sshUsername`（email local-part，后续可接 CSV）与 `password`（请求可选，否则随机生成）
  6. 生成 ID：`app-` + 8 位 hex；`audit_status = APPROVED`；`expire_at = start + days`
  7. 返回 201 + ApplicationVO（凭据在 ACTIVE 前不对外展示完整密码）
```

#### GET `/api/applications/mine`

**需要 JWT**

```
查询参数：
  - status    : String（可选）
  - page      : int，默认 1
  - page_size : int，默认 20，最大 100

处理逻辑：
  1. 解析 JWT，获取当前用户 ID
  2. JPA 动态查询：WHERE user_id = ? [AND audit_status = ?]
  3. 分页排序：updated_at DESC
  4. 返回 200 OK + PageResult<ApplicationVO>
  5. ApplicationVO 不含 user_email 字段
```

## 8. 幂等性实现（Redis 方案）

> 申请提交接口使用 Redis 缓存 `Idempotency-Key`，避免重复写入数据库。

```java
// com.zerodtree.gsad.domain.application.ApplicationService
public ApplicationVO createApplication(CreateApplicationRequest req,
                                       String idempotencyKey,
                                       Long userId) {
    if (idempotencyKey != null) {
        // 1. 先查 Redis 缓存（Key TTL = 24h）
        String cachedId = redisTemplate.opsForValue().get("idempotent:" + idempotencyKey);
        if (cachedId != null) {
            return applicationRepository.findById(cachedId)
                .map(this::toVO)
                .orElseThrow(() -> new BusinessException(ErrorCode.NOT_FOUND, "申请不存在"));
        }
    }

    // 2. 正常创建
    Application app = buildApplication(req, userId);
    applicationRepository.save(app);

    // 3. 写入 Redis（同时作为数据库唯一约束的补充）
    if (idempotencyKey != null) {
        redisTemplate.opsForValue().set("idempotent:" + idempotencyKey, app.getId(),
            Duration.ofHours(24));
    }
    return toVO(app);
}
```

> **回退方案**：若不引入 Redis，可直接依赖 `t_application.idempotency_key` 的数据库唯一约束，捕获 `DataIntegrityViolationException` 后查询并返回原记录。

```
请求体：ServerReportRequest（hostname, resourceLevel, summary, gpus）
处理逻辑：
  1. UPSERT t_server，更新 metrics / ONLINE
  2. 返回 pendingGrants（APPROVED + 本机 server_id）与 pendingRevokes（ACTIVE 且已到期）
```

## 14. 部署建议

### 14.1 环境变量清单

| 变量名 | 说明 | 是否必填 |
| :--- | :--- | :--- |
| `DB_HOST` | PostgreSQL 主机 | 是 |
| `DB_USER` | 数据库用户名 | 是 |
| `DB_PASSWORD` | 数据库密码 | 是 |
| `REDIS_HOST` | Redis 主机 | 是 |
| `REDIS_PASSWORD` | Redis 密码 | 否 |
| `JWT_SECRET` | JWT 签名密钥（≥32字符随机串） | 是 |
| `AGENT_PSK` | Agent 上报接口预共享密钥 | 是 |

### 14.2 Docker Compose 示例

```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: gsad
      POSTGRES_USER: gsad
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD}

  backend:
    image: gsad-backend:latest
    ports:
      - "8080:8080"
    environment:
      DB_HOST: postgres
      DB_USER: gsad
      DB_PASSWORD: ${DB_PASSWORD}
      REDIS_HOST: redis
      REDIS_PASSWORD: ${REDIS_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
      AGENT_PSK: ${AGENT_PSK}
    depends_on:
      - postgres
      - redis

volumes:
  postgres_data:
```
